## 비교 4: 슬롯 매칭 가능성

### 변환 시 예상 효과 차이

AUTO(클론)로 수상 PDF를 변환하면:

| 문항 | 예상 결과 |
|------|---------|
| 3, 4, 5, 19번 | 슬롯 수 동일 → 정상 매칭 가능 |
| 6번 (AUTO=22슬롯, GOLD=9) | 슬롯이 너무 많아 빈 슬롯 22개 남음 |
| 7번 (AUTO=16, GOLD=1) | 마찬가지 |
| 21~24번 | **슬롯 자체 없음 → 수식이 아예 채워지지 않음** |

### 포맷 불일치가 미치는 영향

- **AUTO의 백틱 수식(`x+1``)**: 파서(pdf_filler)가 읽어서 교체하려면 같은 형식으로 변환된 OCR 결과와 매칭돼야 함
- **GOLD의 HWP 내장수식(`x+1`)**: 더 유연하게 매칭됨
- 실제 실험(변환 결과): **AUTO로 변환 시 매칭률 0.9%(2/224)** vs 경신여고-확통(같은 과목 클론)의 경우 11.6%

### 핵심 문제 요약

1. **문항 범위 불일치**: 수1(20문항) → 수상(24문항) — 21~24번(43슬롯) 공백
2. **슬롯 수 불일치**: 같은 번호 문항이라도 복잡도가 달라 슬롯 수 차이가 큼 (6번: 22→9개)
3. **포맷 세대 불일치**: 구형 백틱 vs 현대 HWP 내장수식 — OCR 파서 매칭 효율 저하

### 강점: 잘 된 부분

1. **학교명 교체 완벽** — clone_template의 핵심 기능은 100% 작동
2. **레이아웃 보존 완벽** — 페이지 크기·여백·단 구조 그대로 복사
3. **헤더/푸터 유지** — 동성고 이름이 정상적으로 표시됨
4. **새 워드초벌 자동 저장** — 다음 동성고 시험에는 자동 재사용 가능

### 한계: 부족한 부분

1. **슬롯 수 불일치(핵심)** — 다른 과목을 기반으로 클론하면 문항별 수식 슬롯 수가 완전히 달라짐. 이를 보완할 방법이 없음
2. **문항 범위 불일치** — 서술형 문항 수가 과목마다 다름(수상은 21~24번, 수1은 19~20번). 4문항(43슬롯)이 통째로 빠짐
3. **수식 포맷 세대 차이** — 구형(백틱) 워드초벌을 기반으로 클론하면 OCR 매칭률이 더 낮아짐
4. **이미지 불일치** — 문제 번호 배경 이미지 등이 원본 시험지에 맞게 되어있지 않음

### 개선 방향

**현재 시스템으로 보강 가능한 부분**:

- `clone_template`에 "과목이 다르면 슬롯 구조가 맞지 않을 수 있습니다. 같은 과목 원본을 선택하세요" 경고 추가
- 목록 표시 시 **같은 과목 우선 정렬** (수상 → 수상, 공수1 → 공수1)
- 목록에 "슬롯 수 예상 일치율" 표시

**결국 사람이 셋업해야 하는 부분**:

- **처음 보는 과목의 워드초벌은 수동 제작 필수** — 클론으로는 슬롯 수/구조를 맞출 수 없음
- 신형(HWP 내장수식) 포맷으로 만들어야 OCR 매칭률이 높아짐
- 문항 범위(선택형 몇 번까지, 서술형 몇 번까지)가 과목마다 달라 구조 자체를 새로 작성해야 함
